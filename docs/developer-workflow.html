Generated App Architecture

Data Flow

AMPS Serverws://amps:9008/amps/json
↓
AmpsTransportService@macro/transports/angular
↓ sowAndSubscribe(topic)
ConflationSubject@macro/rxutils — 100ms batching
↓ updateRows$.next(batch)
MacroAngularGrid@macro/macro-angular-grid — AG Grid 35

Key Patterns

  • sowAndSubscribe() — atomic SOW + live updates
  • sowComplete promise — know when initial batch done
  • Transport handles group_begin/end internally
  • Composite keys auto-detected from config
  • Signals for reactive state (Angular 21)
  • Zoneless change detection
  • Dark mode by default

AMPS Transport Features

  • sow() returns clean Promise<[]>
  • deltaSubscribe() for bandwidth efficiency
  • sowDelete() / sowDeleteByKeys()
  • Self-signed cert support (wss://)

Getting Started

1

Build the MCP server

npx nx build macro-mcp — builds once, tools are ready

2

MCP config is automatic

.mcp.json in repo root configures Claude Code / VS Code. For remote: macro-mcp-agent on any server.

3

Talk to your AI assistant

"Import the Figma app at C:/Downloads/my-app.zip as my-app on port 4205"

"Explore AMPS at http://amps:8085/amps/instance/config.xml, topic rates/marketData"

"Create a React app called fx-blotter from AMPS topic fx-rates on port 4207"

4

Run and launch

npm run start:my-app then npm run launch — app appears in OpenFin

Works with Claude Code, VS Code GitHub Copilot, WebStorm AI — any MCP-compatible client
